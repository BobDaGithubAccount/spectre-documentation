Plugin crash course

Plugins are JAR files that are loaded by the JVM class loader, scanning for annotations like @EventHandler for event methods, implementations of certain interfaces or abstract classes (Like JavaPlugin or Listener) + respective overrides (like the onEnable() entry point) and data files, loading it into memory. They can be used to add custom game logic, packet handling and with asset packages bundled to the client, custom items and entities. The beauty of plugins is that they are completely server side, allowing for complete control for the developer.

Plugins are located in the /plugins folder in the server's root directory. Exported plugins should be Java binaries stored as a JAR, and should be in the plugin folder. Their special data folder should be located at /plugins/"plugin_name", and the jar at /plugins/"plugin_name".jar
Plugin jar datastructure should be as follows:
DataStructure:
./plugin.txt
./path.to.package.with.main.class/MainClass
The rest is up to you! It's just Java!

Plugin.txt

This file stores data about the plugin that is scanned upon load. The current, and required data points are all in the following :
name: TestPlugin
author: Jephacake
description: This is a test plugin!
version: 1.0
spectre_version: 1.0
main: (main class location) pathtopackagewithmainclass.MainClass
